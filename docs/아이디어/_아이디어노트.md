# PivotGrid — 아이디어노트



## 한 줄 요약

Microsoft PivotViewer의 로컬/웹 대체 앱 — CSV 드래그앤드롭하면 바로 작동하는 시각적 데이터 탐색 도구.

---

## 문제와 해결 방향

**문제 1**: PivotViewer(Silverlight)는 단종됨. 대규모 데이터를 시각적으로 탐색하는 도구가 없거나 상용(Zegami 등)이라 개인 사용에 부적합.

**문제 2 (핵심)**: 원본 PivotViewer는 시각화는 혁신적이었지만, **데이터 준비가 너무 어려웠음** — CXML 작성, DZI 이미지 타일링, facet 수동 정의 등 하나의 컬렉션 만드는데 몇 시간~며칠.

**해결**:
```
PivotViewer: 멋진 시각화, 근데 데이터 준비가 지옥
PivotGrid:   CSV 하나 드래그앤드롭하면 바로 작동
```

---

## 핵심 차별점: 데이터 준비를 제로에 가깝게

| 입력 | 자동 처리 | 결과 |
|------|----------|------|
| CSV/Excel 드롭 | 컬럼 자동 인식 → facet 생성 | 즉시 격자+필터 |
| JSON 드롭 | 키 자동 인식 | 즉시 격자+필터 |
| 이미지 폴더 드롭 | EXIF/파일명 자동 추출 | 이미지 격자+필터 |
| URL (Google Scholar 등) | 스크래핑 | 논문 격자+필터 |
| gabqca/GT-analyzer 결과 | API 연동 | 분석 결과 격자+필터 |

### 데이터 → 시각화 자동 흐름

```
Step 1: 파일 드롭 (CSV, Excel, JSON)
           │
Step 2: 자동 분석
           ├─ 컬럼 타입 감지 (텍스트/숫자/날짜/카테고리)
           ├─ 카테고리 컬럼 → 필터(facet) 자동 생성
           ├─ 숫자 컬럼 → 범위 슬라이더 자동 생성
           ├─ 텍스트 컬럼 → 카드 제목/본문으로 매핑
           └─ 이미지 URL 컬럼 → 썸네일 자동 표시
           │
Step 3: 즉시 격자 표시 (수정 없이 바로)
           │
Step 4: 사용자가 원하면 커스터마이즈
           ├─ 어떤 컬럼을 facet으로 쓸지 선택
           ├─ 카드에 보여줄 정보 선택
           └─ 정렬/그룹핑 기준 선택
```

## 대상 사용자

- 본인 (연구 데이터, 비블리오메트릭스, QCA 사례 탐색)
- 질적/양적 연구자
- 데이터 컬렉션을 시각적으로 탐색하고 싶은 누구든

---

## 핵심 기능

1. **CSV 드롭 → 즉시 격자**: 파일 하나로 바로 시각화
2. **자동 facet 생성**: 컬럼 타입 감지 → 필터 자동 생성
3. **격자 뷰**: 아이템을 격자(grid)로 배치, 카드/썸네일 형태
4. **다면 필터(Faceted Filter)**: 조건 변경 시 아이템이 애니메이션으로 재그룹핑
5. **정렬/그룹핑**: 속성별 정렬, 그룹핑 (시각적 클러스터)
6. **카드 상세**: 클릭 → 모달로 상세 정보 표시
7. **관리 용이**: CSV 하나로 데이터 관리

## PivotViewer 핵심 UX 재현

```
데이터 로드 → 격자 배치 (모든 아이템 한눈에)
    │
    ├─ 필터 변경 → 부드러운 애니메이션으로 재배치
    ├─ 정렬 변경 → 아이템 위치가 스르르 이동
    ├─ 그룹핑 → 시각적 클러스터 형성
    └─ 클릭 → 개별 아이템 상세 (모달)
```

## 기술적 접근 — 두 가지 선택지

| 접근 | 설명 | 장점 | 단점 |
|------|------|------|------|
| A. **Canvas/PixiJS** | 모든 아이템을 Canvas에 직접 렌더링 | 고성능, 수만 개 가능 | 텍스트 렌더링 어려움, 접근성 낮음 |
| B. **DOM + FLIP** | CSS Grid + Framer Motion으로 애니메이션 | 구현 쉬움, 텍스트 자연스러움 | 수백 개까지만 (수천 개 성능 문제) |

개인 데이터 규모(논문 50편, 사례 수백 건)라면 **B(DOM + FLIP)**로 시작 충분. 이후 수천 개 필요하면 Canvas로 전환.

### 난이도 재평가 (데이터 준비 자동화가 핵심일 때)

| 기능 | 난이도 | 비고 |
|------|--------|------|
| **CSV 자동 파싱 + facet 생성** | **중 (핵심)** | 컬럼 타입 감지, 카테고리 추출 |
| 격자 배치 | 낮음 | CSS Grid |
| 필터링 후 애니메이션 재배치 | 중 | Framer Motion FLIP |
| 카드 클릭 → 모달 상세 | 낮음 | 표준 UI |
| Deep Zoom | 높음 (Phase 5+) | 카드 모달로 대체 가능 |

## 기술 스택 (구상)

| 구성 | 기술 | 이유 |
|------|------|------|
| 프레임워크 | Next.js (chad-app-template-light) | 경량 템플릿 |
| 애니메이션 | Framer Motion (FLIP) | DOM 기반 부드러운 재배치 |
| 데이터 로직 | D3.js 또는 crossfilter | 필터/정렬/그룹핑 |
| 백엔드 | FastAPI | 파일 업로드, 데이터 파싱 |
| DB | SQLite 또는 JSON 파일 | 로컬 경량 |

## 구현 순서 (구상)

```
Phase 1: CSV 드롭 → 자동 facet → 격자 표시 + 필터  ← 이것만으로 유용
Phase 2: 필터 변경 시 애니메이션 재배치
Phase 3: 정렬/그룹핑
Phase 4: 카드 클릭 → 상세 모달
Phase 5: 이미지 지원, 다양한 데이터 소스
Phase 6: Deep Zoom (필요 시)
```
